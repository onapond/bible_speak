# Commit Skill

변경사항을 분석하고 안전하게 커밋한다.

## 단계

### 1. 변경사항 확인
- `git status`로 변경/추가/삭제된 파일 확인
- `git diff`로 변경 내용 분석
- 변경사항이 없으면 "커밋할 변경사항이 없습니다" 안내 후 종료

### 2. 보안 검사
- `.env`, 자격 증명 파일, API 키가 포함된 파일은 절대 스테이징하지 않는다
- 의심되는 파일 발견 시 사용자에게 경고

### 3. 커밋 메시지 생성
- 변경 내용을 분석하여 Conventional Commits 형식으로 작성
  - `feat:` 새 기능
  - `fix:` 버그 수정
  - `chore:` 유지보수
  - `docs:` 문서
  - `refactor:` 리팩토링
  - `style:` UI/스타일
  - `perf:` 성능 개선
- 한국어/영어 상관없이 내용 기반으로 작성
- 1~2문장으로 간결하게

### 4. 커밋 실행
- 관련 파일만 선택적으로 `git add`
- 커밋 메시지는 반드시 HEREDOC 방식으로 전달 (쉘 이스케이프 문제 방지)
```bash
git commit -m "$(cat <<'EOF'
메시지 내용
EOF
)"
```
- 괄호, 한국어, 특수문자가 있어도 HEREDOC이면 안전

### 5. 결과 확인
- `git status`로 커밋 성공 확인
- 커밋 해시와 메시지 출력

## 주의사항
- `git add -A` 또는 `git add .` 사용 금지 — 파일 지정하여 추가
- 사용자가 명시적으로 요청하지 않는 한 push하지 않는다
- 기존 커밋을 amend하지 않는다 — 항상 새 커밋 생성
- Windows 환경에서는 쉘 이스케이프에 주의 (HEREDOC 필수)
