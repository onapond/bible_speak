# Performance Optimize Skill

벤치마크 기반으로 성능을 측정하고 최적화한다. 추측이 아닌 데이터로 판단한다.

## 단계

### 1. 최적화 대상 확인
- 사용자에게 느린 부분, 목표 성능을 확인한다
- 웹: Lighthouse, Core Web Vitals (LCP, FID, CLS)
- 앱: 로딩 시간, 프레임 드롭, 메모리 사용량
- API: 응답 시간, 쿼리 수

### 2. 베이스라인 측정
- 현재 성능을 측정하여 기록한다
- 가능하면 타이밍 코드를 임시 추가하여 각 단계별 소요 시간을 측정한다
- 측정 결과를 사용자에게 보고한다

### 3. 병목 식별
- 측정 데이터를 기반으로 병목 지점을 순위별로 정리한다
- 가장 느린 부분부터 최적화한다 (80/20 법칙)
- 추측으로 최적화하지 않는다

### 4. 최적화 루프 (반복)
각 병목에 대해:
1. **수정**: 하나의 최적화만 적용
2. **재측정**: 같은 방법으로 성능 측정
3. **판단**:
   - 15% 이상 개선 → 유지
   - 15% 미만 개선 → 되돌리고 다른 접근 시도
4. **기록**: 시도한 것, 결과, 유지/되돌림 여부

같은 병목에서 3번 시도해도 개선이 없으면 다음 병목으로 이동한다.

### 5. 결과 보고
- 최적화 전후 비교표 출력
- 각 최적화별 개선 수치
- 목표 달성 여부

## 금지 사항
- 측정 없이 최적화 금지
- 한 번에 여러 최적화를 동시 적용 금지 (효과 측정 불가)
- 타임아웃 늘리기, 리트라이 추가 등 회피성 최적화 금지
- 코드 가독성을 심각하게 훼손하는 최적화 금지
