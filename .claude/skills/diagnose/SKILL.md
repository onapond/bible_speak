# Diagnose Skill

버그나 성능 문제의 근본 원인을 먼저 분석한 후 수정한다. 증상만 보고 즉시 코드를 고치지 않는다.

## 단계

### 1. 문제 수집
- 사용자에게 증상, 에러 메시지, 재현 조건 확인
- 관련 로그, 콘솔 출력, 스크린샷이 있는지 확인

### 2. 근본 원인 조사
- 코드를 수정하기 **전에** 원인을 파악한다
- 관련 코드 경로를 추적하며 읽는다
- 의심되는 원인 목록을 만들고 각각의 근거를 제시한다
- 필요하면 타이밍 로그, 디버그 출력을 임시 추가하여 측정한다

### 3. 진단 보고
사용자에게 다음을 보고한다:
- **근본 원인**: 문제가 발생하는 정확한 위치와 이유
- **근거**: 로그, 코드 경로, 측정 데이터 등 증거
- **영향 범위**: 이 문제가 다른 곳에도 영향을 주는지
- **수정 방안**: 가능한 해결책 (1개 이상)

### 4. 수정
- 사용자가 진단을 확인한 **후에만** 코드를 수정한다
- 수정 범위는 근본 원인에만 집중한다
- 관련 없는 코드를 같이 고치지 않는다

### 5. 검증
- 수정 후 문제가 해결되었는지 확인한다
- 같은 증상이 재현되지 않는지 확인한다

## 금지 사항
- 원인 파악 전에 코드 수정 금지
- 추측만으로 수정 금지 — 반드시 증거 기반
- 타임아웃 값 변경, 리트라이 추가 등 증상 회피 금지
- 한 번에 여러 곳을 동시에 수정하지 않는다 (원인 특정 불가)
