# 바이블스픽 UX 개선 계획

**작성일**: 2026-01-29
**목표**: 사용자 경험 극대화 및 이탈률 감소

---

## 요약

바이블스픽 앱은 3단계 학습, 발음 피드백, 소셜 기능 등 강력한 기능을 갖추고 있으나, 다음 영역에서 사용자 경험 개선이 필요합니다:

1. **정보 부족** - 핵심 결정 시점(온보딩, 그룹 선택, 단계 진행)에서 설명 부재
2. **불명확한 에러 상태** - 실패가 조용히 처리되어 사용자가 상황 파악 불가
3. **복잡한 메인 메뉴** - 11개 옵션이 계층 구조 없이 나열
4. **접근성 낮은 피드백** - IPA 기호, 복잡한 음소 분석이 설명 없이 제공
5. **맥락 부재** - 로딩 상태, 빈 상태에서 안내 부족

---

## 1. 온보딩 및 첫 사용자 경험

### 문제점

| 문제 | 영향 | 심각도 |
|------|------|--------|
| 그룹 기능 설명 없이 그룹 선택 요구 | 사용자 혼란, 이탈 | 🔴 높음 |
| 온보딩 후 11개 메뉴에 바로 노출 | 인지 과부하 | 🔴 높음 |
| 권한 요청 맥락 없음 (마이크 등) | 권한 거부 → 기능 불가 | 🟡 중간 |
| 프로필 설정 완료 후 안내 없음 | "다음 뭐하지?" 혼란 | 🟡 중간 |

### 개선 방향

1. **온보딩에 그룹 설명 추가** - 그룹이 뭔지, 왜 필요한지 사전 설명
2. **첫 화면 간소화** - 핵심 기능(학습하기)만 강조, 나머지는 점진적 노출
3. **첫 학습 가이드** - 3단계 학습 시스템 튜토리얼 제공
4. **권한 요청 사전 설명** - 왜 필요한지 먼저 설명 후 요청

### 구현 계획

```
[x] 온보딩 페이지에 "그룹과 함께 성장하세요" 슬라이드 추가
[x] 프로필 설정 완료 후 "학습 시작하기" 가이드 다이얼로그
[x] 첫 학습 시 3단계 설명 오버레이
[ ] 권한 요청 전 설명 다이얼로그 (마이크, 알림)
```

## 2. 메인 메뉴 복잡도

### 문제점

| 문제 | 영향 | 심각도 |
|------|------|--------|
| 11개 메뉴 항목 동일 크기로 나열 | 핵심 기능 찾기 어려움 | 🔴 높음 |
| 학습 vs 소셜 vs 관리 구분 없음 | 인지 부하 | 🔴 높음 |
| 설정이 모달 바텀시트에 숨겨짐 | 발견성 낮음 | 🟡 중간 |
| "복습" vs "암송 연습" 차이 불명확 | 기능 혼동 | 🟡 중간 |

### 개선 방향

1. **핵심 기능 강조** - "오늘의 학습" CTA 버튼을 최상단에 배치
2. **카테고리 그룹화** - 학습 / 소셜 / 내 정보 섹션 분리
3. **빠른 액션** - 자주 쓰는 기능 1탭 접근
4. **메뉴 설명 개선** - 각 메뉴의 목적을 명확히

### 구현 계획

```
[x] "오늘의 학습 시작" 큰 버튼 추가 (연속 학습 + 복습 예정 표시)
[x] 메뉴 그룹화: 학습 (암송, 단어, 퀴즈, 복습) / 소셜 (그룹, 친구) / 내 정보 (통계, 업적, 샵, 프로필)
[x] 메뉴 부제목 명확화: "복습" → "복습 (오늘 3개)"
[ ] 설정 아이콘을 앱바에 노출
```

## 3. 학습 플로우 (VersePracticeScreen)

### 문제점

| 문제 | 영향 | 심각도 |
|------|------|--------|
| 3단계 시스템 설명 없음 | 사용자가 목표 모름 | 🔴 높음 |
| 통과 기준 불명확 | "왜 안 넘어가지?" 혼란 | 🔴 높음 |
| 이전 점수와 비교 불가 | 성장 체감 어려움 | 🟡 중간 |
| 다음 구절로 자동 이동 없음 | 수동 탐색 필요 | 🟡 중간 |
| AI 피드백 로딩 중 안내 없음 | 사용자가 기다리는지 모름 | 🟡 중간 |

### 개선 방향

1. **단계 진행 시각화** - 각 단계의 목표와 현재 상태 명시
2. **점수 비교** - "이전: 72% → 이번: 85% 🎉" 형태로 개선 표시
3. **자동 진행 제안** - 통과 시 "다음 구절로?" 제안
4. **피드백 로딩 표시** - "AI 코치가 분석 중..." 명시

### 구현 계획

```
[x] 단계별 통과 기준 상시 표시 (Stage 1: 70%, Stage 2: 75%, Stage 3: 80%)
[x] 이전 최고 점수 표시 및 개선 시 축하 애니메이션
[x] 통과 시 "다음 구절 학습하기" 버튼 자동 표시
[x] AI 피드백 로딩 상태 명시적 표시
[ ] 연속 학습 모드 (자동으로 다음 구절 로드)
```

## 4. 발음 피드백 접근성

### 문제점

| 문제 | 영향 | 심각도 |
|------|------|--------|
| IPA 기호 설명 없음 (/ð/, /θ/) | 피드백 이해 불가 | 🔴 높음 |
| 음소별 개선 방법 안내 없음 | 액션 불가 | 🔴 높음 |
| 단어 클릭 시 팝업만 제공 | 인라인 도움 없음 | 🟡 중간 |
| 정확한 발음 들어볼 수 없음 | 참고 자료 부재 | 🟡 중간 |

### 개선 방향

1. **한글 발음 가이드 강화** - IPA 옆에 한글 발음 힌트 크게 표시
2. **발음 팁 즉시 표시** - 낮은 점수 음소에 자동으로 팁 노출
3. **참조 발음 재생** - 정확한 발음을 들을 수 있는 버튼 추가
4. **개선 가이드 제공** - "입 모양을 이렇게 해보세요" 등 구체적 안내

### 구현 계획

```
[x] 음소 점수 바에 한글 발음 가이드 크게 표시
[x] 80% 미만 음소에 발음 팁 자동 펼침
[x] "이 소리 듣기" TTS 버튼 추가
[ ] 입 모양 이미지/애니메이션 추가
```

## 5. 로딩 및 에러 상태

### 문제점

| 문제 | 영향 | 심각도 |
|------|------|--------|
| 타임아웃 시 조용한 실패 | 사용자가 상황 모름 | 🔴 높음 |
| 로딩 중 무엇을 기다리는지 모름 | 불안감 | 🟡 중간 |
| 에러 메시지에 기술 용어 노출 | 사용자 혼란 | 🟡 중간 |
| 빈 상태에서 안내 없음 | 다음 액션 모름 | 🟡 중간 |
| 재시도 가능 여부 불명확 | 반복 실패 | 🟡 중간 |

### 개선 방향

1. **명시적 로딩 메시지** - "성경 구절을 불러오는 중 (보통 2-3초)"
2. **친절한 에러 메시지** - 기술 용어 제거, 해결 방법 안내
3. **빈 상태 가이드** - "아직 학습 기록이 없어요. 첫 암송을 시작해볼까요?"
4. **네트워크 상태 표시** - 오프라인일 때 명확히 안내

### 구현 계획

```
[x] 로딩 화면에 설명 텍스트 추가
[x] 에러 메시지 사용자 친화적으로 변경
[x] 빈 상태 컴포넌트 생성 (일러스트 + 안내 + CTA)
[x] 네트워크 에러 시 "인터넷 연결을 확인해주세요" 표시
[ ] 오프라인 모드 명확화 (어떤 기능 사용 가능한지)
```

|------|------|--------|
| 그룹 챌린지 기여 방법 불명확 | 참여 저조 | 🟡 중간 |
| 찌르기(Nudge) 한도 사전 안내 없음 | 실패 시 당황 | 🟡 중간 |
| 친구 배틀이 뭔지 설명 없음 | 기능 미사용 | 🟡 중간 |
| 빈 멤버 목록에서 안내 없음 | "어떻게 초대?" | 🟡 중간 |

3. **기능 소개 툴팁** - 첫 접근 시 간단한 설명
4. **빈 상태에 액션 제안** - "그룹 초대 링크 복사하기"

### 구현 계획

```

[x] 찌르기 버튼에 남은 횟수 표시
[ ] 친구 배틀 설명 다이얼로그
```
